	•	swipe_left
	•	swipe_right

session_started → filters_set → second_user_joined → first swipe → match_created

	3.	Restaurant fetching

Use Google Places API.

Inputs (already exist):
	•	latitude/longitude
	•	radius
	•	price level
	•	vibe selections

	4.	Filtering

Before ranking, only include restaurants where:
	•	rating >= 4.2
	•	review_count >= 75
	•	business_status is operational

	5.	Scoring function

Score each restaurant using:
rating weighted at 40 percent
log(review_count) weighted at 25 percent
distance_score weighted at 15 percent (inverse of distance)
vibe_score weighted at 20 percent (keyword match count)
	6.	Vibe mapping (soft ranking boost)

Map vibes to keywords:

Cozy → cozy, intimate, cafe, wine bar
Trendy → trendy, modern, popular
Romantic → romantic, date night, italian, steakhouse
Casual → casual, quick bite, local
Lively → bar, brunch, music, busy

Rules:
	•	combine multiple vibes (union of keywords)
	•	apply as ranking boost, not a strict filter
	•	if results are too few, relax vibe weighting only (do not change radius or price filters)

	7.	Deck generation

	•	sort restaurants by score descending
	•	return top 20 to 30 results
	•	cache results per session

	8.	Feedback loop

On match_created:
	•	store restaurant_id in events

Use this to slightly boost ranking of restaurants with higher historical match frequency.
